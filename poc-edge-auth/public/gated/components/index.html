Button Component

A versatile button component with multiple variants and advanced features.

Basic Usage:

<astro-button variant="primary">Click Me</astro-button>

Advanced Example:

<astro-button variant="primary" size="large" icon="rocket" loading={isLoading} disabled={isDisabled} onClick={handleClick} data-analytics="cta-main" > Launch Application </astro-button>

Complete API:

Property Type Default Description
variant string primary primary | secondary | danger | ghost | link
size string medium small | medium | large | xl
icon string null Icon name from icon library
loading boolean false Show loading spinner
fullWidth boolean false Expand to full container width

üîß Implementation Notes:

  • Use loading state for async operations
  • Combine with astro-tooltip for additional context
  • Supports keyboard navigation (Enter/Space)
  • ARIA compliant with proper roles and labels

Card Component

A flexible container component with extensive customization options.

Basic Usage:

<astro-card title="My Card">Content here</astro-card>

Advanced Example:

<astro-card title="Mission Status" subtitle="Real-time monitoring" headerIcon="satellite" elevation={2} interactive onClick={handleCardClick} > <astro-card-media src="/mission-photo.jpg" alt="Mission photo" aspectRatio="16:9" /> <astro-card-content> <p>Mission parameters...</p> </astro-card-content> <astro-card-actions> <astro-button variant="primary">View Details</astro-button> <astro-button variant="ghost">Share</astro-button> </astro-card-actions> </astro-card>

Complete API:

Property Type Default Description
title string null Card header title
subtitle string null Secondary header text
elevation number 1 Shadow depth (0-5)
interactive boolean false Add hover effects
collapsible boolean false Allow expand/collapse

üöÄ Pro Tips:

  • Use elevation={0} for nested cards
  • Combine multiple cards in a astro-grid for layouts
  • Card actions support up to 3 buttons optimally
  • Use collapsible for long content sections

Alert Component

Comprehensive alert system with animations and advanced features.

Basic Usage:

<astro-alert type="info">Information message</astro-alert>

Advanced Example:

<astro-alert type="warning" title="System Maintenance" dismissible icon="warning" onDismiss={handleDismiss} autoClose={5000} position="top-right" animate="slide" > Scheduled maintenance window: 2:00 AM - 4:00 AM UTC <astro-alert-action> <a href="/status">View Status Page</a> </astro-alert-action> </astro-alert>

Toast Notifications:

// JavaScript API for toast alerts import { showAlert } from '@astro/alerts'; showAlert({ type: 'success', message: 'Operation completed successfully', duration: 3000, position: 'bottom-center' });

üéØ Best Practices:

  • Use error type sparingly for critical issues only
  • Set autoClose for transient notifications
  • Include action links for actionable alerts
  • Position based on importance and user flow
  • Test with screen readers for accessibility
